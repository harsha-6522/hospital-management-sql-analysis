# Hospital Management Data Analysis using MYSQL

## Project Overview
This project analyzes a hospital management dataset using SQL.  
The dataset contains multiple related tables including patients, doctors, appointments, treatments, and billing records.  
The goal of this project is to explore relational data, perform SQL queries, and generate business insights.

## Tools Used
- MySQL
- SQL (JOIN, GROUP BY, Aggregate Functions)
- Relational Database Analysis

## Dataset Tables
The dataset contains the following tables:

- patients
- doctors
- appointments
- treatments
- billing

## Analysis Performed

### Data Exploration
- Previewed tables using SELECT
- Checked record counts using COUNT()
- Examined table structure using DESCRIBE

### Patient Analysis
- Patients by gender
- Patients by insurance provider

### Doctor Analysis
- Doctors by specialization
- Doctors with most appointments

### Appointment Analysis
- Appointment status distribution
- Number of appointments per doctor

### Treatment Analysis
- Most common treatments

### Billing / Revenue Analysis
- Total revenue generated
- Revenue by payment method
- Revenue by treatment type

### JOIN Queries
- Doctors and appointments join
- Treatments and billing join
- Patients, doctors, and appointments multi-table join

## Key Insights

- Male patients are slightly higher than female patients.
- Pediatrics has the highest number of doctors.
- No-show appointments are common in the dataset.
- Chemotherapy generated the highest revenue.
- Payment methods vary, with some contributing more revenue.
- Multi-table joins help understand relationships between patients, doctors, and treatments.
